The One Legitimate Grid of Items

Every other archetype in this catalog treats a uniform grid of identical boxes as an anti-pattern to design around. This one exists because sometimes the content really is a flat set of comparable items — and the honest answer isn't a card grid, it's a table.

masthead · 1-12
Quiet identity band — archetype name, entry count, category count. Metadata about the index, never a KPI dashboard opener.
rail · 1-3
Optional alphabetic/category jump-nav. Present only once the index is long enough to need it — omit it below roughly 30 entries.
body · 4-12
The index itself: a real table. Alignment and hairline rules are the ONLY structuring device — no borders, fills, shadows, or rounded corners on individual entries.
Alignment is the argument. If a row needs a border to be told apart from its neighbor, the rule failed — not the row.

A card catalogue's discipline comes from three things working together: real table semantics (rows and columns, not divs pretending to be cards), a single sort order the reader can trust, and rules or whitespace instead of borders to separate entries. Take any one of the three away and it drifts back toward the uniform-card-grid anti-pattern this catalog exists to name and avoid everywhere else.

Use this when you have many genuinely comparable items and no single one deserves to dominate: a glossary of terms, a directory of staff or vendors, an index of API endpoints, a bibliography, or a component library's catalog page.

Reading This Archetype

When Not to Use This Archetype

As soon as items have genuinely unequal importance — one flagship case study among five minor ones, one finding that matters more than the rest — reach for magazine-well (size-based dominance) or asymmetric-split instead. This archetype has no mechanism for hierarchy by size, only by position and rule-weight.

The Rule That Prevents a Card-Grid Relapse

No entry gets a border, background fill, shadow, or rounded corner — including a "featured" one. A pinned or emphasized row earns a heavier rule (see the Card / List-Heavy tab), never a box. The moment a row needs its own container to read as distinct, a card grid is creeping back in.

Editorial Devices This Archetype Favors

Category or letter group headers as rule-based section breaks, monospace for any field that's a code or number so digits line up, and a jump-nav rail once the list is long enough to need one. No pull quotes, no callouts, no drop caps — those are narrative devices, and this is reference material.
